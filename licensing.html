GS1 Digital Licensing

GS1 Licensing provides the root of trust for the globally unique trade identification systems. GS1 Digital Licenses are digital representations of the existing GS1 Licenses described in . These Digital licenses can be verified by machines in B2B transactions. Why might digital verification of GS1 Licenses be beneficial?

  • To prove a GS1 MO is authentic within the GS1 system for issuing licenses
  • To prove a MC or MO is authorized to create specific GS1 ID Keys
  • To establish provenance over GS1 ID Keys created by MC or MO
  • To establish provenance over data about GS1 ID Keys created by MC or MO

Digital Licensing in a Nutshell

A Digital GS1 Prefix License is issued by GO to a specific MO. It authorizes them to issue GCPs and GS1 ID Key Licenses from a specific GS1 Prefix value. GS1 Prefix License values are typically three digit numbers, but can range from two to seven digits with details in the General Specification Section 1.4.2. For example, GS1 New Zealand is issued prefixes 940 - 949.

A Digital GCP is issued from an MO to a MC to authorize them to issue GS1 ID Keys from a specific number range. GCPs have different lengths depending on the needs of the MC. GCPs range from four digits to twelve digits. They always begin with (but are at least one digit longer than) the a valid GS1 Prefix License value. For details see General Specification Section 1.4.4. The fewer digits a GCP has, the more individual GS1 ID Keys it can create. For example, a eleven digit GCP can only create a few GTINs while a 7 digit GCP can create many thousands of GTINs. (TODO put a link to some data here that Andrew Provides)

A Digital GS1 ID Key License authorizes a MC to use a single GS1 ID Key for their business operations. GS1 ID Key Licenses are often used by small businesses to generate a few GTIN numbers for their products. They are also used to issue GLN numbers to identify organizations in the global supply chain that participate, but are not creating product, shipments or assets.

Digital License Credentials Overview

GS1 Digital Licenses use the W3C Verifiable Credentials Data Model Version 2.0. The VCDM credential format uses JSON-LD to encapsulate the details of a GS1 License in the format that is well understood worldwide. licenseValue

Every license credential includes a licenseValue. The license credential type is usually sufficient to define, together with the licenseValue, the “space” or “range” of the GS1 identification system granted to the licensee. The exception to this is any license credential that has to do with a GS1 identification key, which requires an identification key type as an additional qualifier. For license credentials that have to do with prefixes, the license value is in normalized GS1 (i.e., non-U.P.C.) form. Where the license credential is within the U.P.C. range, the alternative license value holds the U.P.C. form. For license credentials that have to do with GS1 identification keys, the license value is the key, including the check digit or check characters if applicable, and excluding the serial component if applicable. When the license is specifically for a GTIN, the license value is the GTIN in its minimum format: GTIN-8, GTIN-12, GTIN-13 (which must not start with a zero), or GTIN-14 (which must start with indicator digit 9). Where the license credential is for a GTIN-8, GTIN-12, or GTIN-13, the alternative license value is the GTIN normalized to 13 digits (by padding with zeros on the left) and with the check digit removed (resulting in a 12-digit string); this value is used when verifying a GTIN with indicator digit 1-8 against a license.

alternativeLicenseValue

Alternative value of the license. For most licenses, the alternative value of the license is the bridge between the U.P.C. system and the rest of the GS1 system: if the license value starts with zero, then the alternative license value is the same as the license value but without the leading zero.

Digital License Credentials Exceptional Cases

This section covers some unusual licensing cases.

GTIN8 Prefix Licenses

a GTIN-8 is a 8 digit GTIN that is used for products where a larger barcode might not fit, such as a lipstick or box of matches. GO can license prefixes to MOs to issue GTIN-8. GTIN-8 prefixes are not issued to MCs, so GTIN-8 are only licensed as individual GS1 ID Keys to MCs. Nevertheless, when handling GS1 License Credentials you may come across a GTIN-8 Prefix License. GTIN-8 Prefix Credentials should always be issued by GO and do not require an extendsCredential property.
