Digital License Validation

Digital License validation is the process of confirming that a specific license is current, authentic, and relevant. Because GS1 Licenses are issued in a federated manor, license validation requires examing the set (chain) of licenses back to the GO root of trust.

Philosophy of License Validation

Digital license validation is the process by which a machine can verify that a license is authentic as part of the GS1 identity system. Generally, this involves validating the chain of licenses starting with the most specific license up to the root license issued by GS1 GO. In addition to checking the fields of the license documents, the documents must link together using the extendsCredential property forming a chain back to the root license. This chain must have properties so that each licenseValue stems from its previous licenseValue. This validation process is presented in detail in the sections below.

For example, in the chain with prefix license 02, GCP 022345 and individual GTIN 02234545645654, we can see the licenseValue of the GTIN starts with the licenseValue of the GCP which starts with the licenseValue from the prefix license, indicating a complete and valid chain

License Validation Rules

The following section contains rules for validation of GS1 Licenses.

These steps must be performed on all GS1 license credentials:
  • The credential document must be a valid VCDM 2.0 credential document (TODO is there a schema)
  • The signature must be valid and from a signing key referenced in the issuer DID
  • The credential must NOT be revoked (see TODO)
  • The credential validFrom must not be in the future

Prefix License Validation Rules

When validating a prefix license, the following additional steps must be performed:

  • The credential document must conform to the JSON schema located at https://id.gs1.org/vc/schema/v1/prefix
  • Issuer field must be a GS1 GO did:web or did:webvh
  • Prefix licenseValue and alternateLicenseValue (if present) must be between 2 and 5 TODO can we add this rule to the schema numeric digits

GCP Validation Rules

When validating a GCP license, the following additional steps must be performed:

  • The credential document must conform to the JSON schema located at https://id.gs1.org/vc/schema/v1/companyprefix
  • The credential referenced by the extendsCredential field must be valid.
  • The GCP licenseValue and alternateLicenseValue (if present) must be between 4 and 11 TODO numeric digits
  • The following checks between the credential and the credential referenced by the extendsCredential
    • The credentialSubject id of the credential referenced by the extendsCredential must match the issuer property of this GCP credential.
    • The GCP licenseValue and alternateLicenseValue (if present) must begin and be at least one digit longer than the licenseValue and alternateLicenseValue of the credential referenced by the extendsCredential

Identification Key License Validation Rules

When validating a Identification Key licsense, the following additional steps must be performed:

  • The credential document must conform to the JSON schema located at https://id.gs1.org/vc/schema/v1/identificationkey
  • The credential referenced by the extendsCredential field must be valid.
  • The following checks between the credential and the credential referenced by the extendsCredential
    • The credentialSubject id of the credential referenced by the extendsCredential must match the issuer property of this ID Key credential.
    • The ID Key credential licenseValue and alternateLicenseValue (if present) must begin with the licenseValue and alternateLicenseValue of the credential referenced by the extendsCredential
